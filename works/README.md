# Works

Your portfolio. Every finished piece gets an entry here as a markdown file named after the piece: `yyyymmdd-title-in-kebab-case.md`.

A "finished piece" is anything that came off the plotter that you consider complete -- even if it didn't turn out how you expected. Failed experiments that taught you something belong here too.

## Writing an entry

Start by writing about the piece however feels natural. Describe what it is, how it came to be, what happened. Some entries might be a paragraph, others might be a page. Let the piece tell you how much it needs.

After you've written, check whether you've covered these:

- **What is it?** Title, date, materials (pen/marker, paper).
- **How was it made?** Number of passes, tools per pass, any code or algorithms involved.
- **What were you trying to do?** The idea, prompt, or impulse that started it.
- **What actually happened?** How the physical result compares to what you imagined. Include a camera capture if available.
- **What did it teach you?** Anything you'd carry forward or do differently.

If something is missing, add it. Don't restructure what you already wrote.

## Naming

Give every piece a title. It doesn't need to be clever -- it just needs to be yours. A titled work is part of a body of work; an untitled file is just output.
